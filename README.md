# Simple Remote Desktop

**Simple Remote Desktop** — это легковесное приложение для удалённого управления рабочим столом Windows. Оно транслирует изображение экрана, системный звук, а также передаёт события клавиатуры и мыши в реальном времени.

Проект написан на **C++** с использованием:
- **DirectX 11 / DXGI** для захвата и отображения экрана
- **Windows Core Audio (WASAPI)** для захвата и воспроизведения звука
- **Windows Compression API (XPRESS)** для сжатия изображения
- **Сокетов (Winsock)** для обмена сообщениями по TCP и UDP

Всё это работает в паре **клиент-сервер**.

---

## Возможности

- **Захват экрана** - передача только изменённых областей с масштабированием и сжатием.
- **Аудио-дупликация** - захват системного звука и отправка на клиент для воспроизведения.
- **Управление мышью** - абсолютное позиционирование, клики (ЛКМ, ПКМ).
- **Управление клавиатурой** - передача нажатий и отпусканий клавиш.
- **Сжатие** - изображения сжимаются алгоритмом XPRESS Huffman для экономии трафика.
- **Автоматическое переподключение** - сервер ждёт нового клиента после разрыва.

---

## Архитектура

Проект разделён на две части: **сервер** (тот, чей экран транслируется) и **клиент** (тот, кто смотрит и управляет).

### Сервер
- **DesktopDuplicator** - захват экрана через `IDXGIOutputDuplication`, выделение изменённых областей, масштабирование до ½ разрешения, сжатие и отправка по TCP.
- **AudioDuplicator** - захват аудио в режиме LOOPBACK через WASAPI, отправка по UDP.
- **KeyboardRepeater** / **MouseRepeater** - приём команд с клиента и эмуляция ввода через `SendInput`.

### Клиент
- **DesktopDemonstrator** - приём сжатых фрагментов экрана, декомпрессия, отображение через D3D11.
- **AudioPlayer** - приём аудиоданных по UDP, буферизация и воспроизведение через WASAPI.
- **KeyboardDuplicator** / **MouseDuplicator** - перехват событий ввода на клиенте и отправка на сервер.

### Сетевая подсистема
- **SocketSender** / **SocketReceiver** - универсальные обёртки для TCP (экран, ввод) и UDP (аудио).

### Потоки
 - Каждый модуль работает в своём потоке.

## Технологии

| Компонент             | Технологии                                                                 |
|-----------------------|----------------------------------------------------------------------------|
| Захват экрана         | Desktop Duplication API + Direct3D 11                          |
| Отображение экрана    | Direct3D 11                                           |
| Аудио                 | WASAPI                                   |
| Сжатие                | Windows Compression API                                   |
| Сеть                  | Winsock 2.2, TCP для управляющих команд, UDP для аудио                   |
| Многопоточность       | std::thread                                                               |

## Сборка и запуск

### Требования
- Windows 10 / 11 (x64 рекомендуется)
- Visual Studio 2022 (или новее) с установленными компонентами:
  - Разработка классических приложений на C++
  - Windows 10 SDK
- Библиотеки (входят в Windows SDK):
  - `d3d11.lib`
  - `cabinet.lib`  
  - `ws2_32.lib`

### Шаги сборки
1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/your-username/simple-remote-desktop.git

2. Откройте файл решения Simple Remote Desktop.sln в Visual Studio.
3. Соберите проект.

### Запуск

1. Запустите сервер на компьютере, экран которого вы хотите транслировать.
2. Запустите клиент на другом компьютере.
3. На клиенте введите IP-адрес сервера.

### Порты (можно изменить в коде):
- **5555** - управляющее соединение (TCP)
- **5556** - клавиатура (TCP)
- **5557** - мышь (TCP)
- **5558** - аудио (UDP)
- **5559** - экран (TCP)
